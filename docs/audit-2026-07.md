## 一、确认的 Bug

### 🔴 高

- ☑ **B1** `console/console.css:47` — `#sites{flex:0 1 auto;overflow:hidden}`：overflow:hidden 把 flex 自动最小宽度归零，窄屏（≤1800px）下全部挤压落在 #sites，尾部站点芯片被静默裁切（不可见、不可点、失败红点不可见），#bar 横向滚动永不触发。修：改 `overflow-x:auto`。

### 🟠 中

- ☑ **B2 群发假成功（静默丢问）** `content/core.js:100-134` — contenteditable 注入失败（execCommand 返回值被丢弃、无兜底校验）时 `_txtBefore` 为空 → Enter 校验循环首轮 `!cur` 即 `ok:true`，圆点变绿但该站从未收到问题。修：注入后 `readText(el)` 为空即返回失败；检查 execCommand 返回值。
- ☑ **B3 切档假成功 toast** `adapters-cn.js:53/139/18/25/96` — 豆包/Kimi/DeepSeek/千问关键控件缺失时静默 return 而非 throw（元宝/智谱/国际站均 throw），runMode 以"不抛错"判成功 → 站点改版后 toast「已切到」但档位未动；switchTier 的 nullTries 捷径也被骗过。修：控件缺失一律 throw（DeepSeek 模式 radio 仅空对话首屏存在，聊天中缺失属正常态，保留静默跳过）。
- ☐ **B4 Gemini 中文界面漏设思考等级** `adapters-intl.js:199` — `/^extended/i` 仅英文，违反「UI 词中英双写」契约；中文界面 think 档模型切成功但等级静默停默认值。修：补中文正则（需真机确认文案），找不到时 throw/toast。
- ☑ **B5 窗口 id 跨重启碰撞** `bg/windows.js:19` — amsWindows/amsConsoleWin/amsComposeWin 持久化，Chrome 重启后窗口 id 重排，按 id 操作只验 type=popup 不验内容 → 陈旧 id 撞上用户 OAuth/站点弹窗时误关/误收编/误聚焦（popup-only 铁律漏洞）。修：`onStartup` 清三个 key。
- ☑ **B6 closeAll/newSessionAll 绕过 serializeOp** `background.js:50/88/89` — 与在途 openTile/sendAll 并发读-改-写 amsWindows（Enter 群发后关 console → console 已关却弹出整批平铺窗）。修：三处包进 serializeOp。
- ☑ **B7 历史 ↑ 覆盖草稿不可恢复** `console/console.js:241` — ↑ 用 history[0] 覆盖未发送草稿并立即落盘；↓ 回 -1 置空不还原。修：进浏览前存草稿变量，↓ 回 -1 还原。
- ☑ **B8 compose 群发无法重试** `console/console.js:139` — lastSend 只在 console send 点击路径赋值，compose 经 bg 发送失败后 retry 恒灰。修：bg 的 sendStart 消息附带 {text,tier}，console 据此填 lastSend。
- ☑ **B9 群发中改分组抹状态** `console/console.js:82` — applyHosts→render() replaceChildren 清掉全部圆点状态/失败 title，失败站脱离 retry 集合。修：render 前快照状态类并恢复。
- ☑ **B10 跨窗口"编辑中"判定失效** `compose.js:38`/`console.js:272` — 窗口失焦后 document.activeElement 不变，双向回填被永久挡住 →「回填并关闭」可用旧文覆盖新编辑。修：判定改 `document.hasFocus() && activeElement === 本框`。

### 🟡 低

- ☑ **B11 IME 保护缺失** `manage.js:31`（nameinput Enter）、`console.js:239/243`（↑↓）— 无 isComposing 保护（console.js:236 的 Enter 有，防护不对称）。修：补 `!e.isComposing`。
- ☑ **B12 raiseConsole 裸查** `bg/windows.js:134` — 裸 tabs.query 不验 type 不认登记 id（全仓唯一漏网），console.html 被开进普通标签时会抢焦用户日常窗口。修：改 `getConsoleWinId()+updateIfPopup`。
- ☐ **B13 多显示器坐标系混用** `bg/windows.js:67` — console 拖到主屏下方副屏时 reserve 变跨屏距离，平铺压成 120px 废格。修：用 console 所在显示器 workArea 做基准（与功能建议 F4 同一改动）。
- ☑ **B14 openConsole 并发双开** `bg/windows.js:91` — 连按 Alt+Q（SW 冷启动时必现）双开 console，先建者成孤儿仍驱动联动。修：in-flight 去重或包 serializeOp。
- ☑ **B15 pill idle 类残留** `pill.js:93` — applyMode 不清 idle，always→handle 后胶囊停 35% 透明。修：applyMode 里 clearTimeout+remove("idle")。

### 📄 文档

- ☑ **D1** `README.md:31` Claude fast 档写「Opus 4.8 + Thinking 关 + Low」，实际为 Sonnet 5 + 思考开 + Medium。
- ☑ **D2** `README.md:19` 仍列已删除的「全部恢复并置顶 / 全部最小化」按钮（现由联动自动完成）。

## 二、发布与安全加固

- ☑ **S1 package.sh 无产物对账**（中）— 只验白名单顶层路径，不对照 manifest/HTML 实际引用（v0.6.0 坏包事故根因未固化为代码）。修：打包后从 manifest/HTML 提取全部引用路径逐一 `zip -sf | grep -qx` 断言。
- ☑ **S2 旧 tag 坏 package.sh**（低）— 远端 v0.5.0/v0.6.0 tag 树内 RUNTIME 缺 i18n.js/_locales，从旧 tag 重打包/重跑 workflow 会复现坏包。修：README 加警示或删废 tag。
- ☑ **S3 Action 未 pin SHA**（低）— release.yml 的 checkout@v4 / action-gh-release@v2 是可变 tag，contents:write 下构成供应链面。修：固定 commit SHA。
- ☐ **S4 tabs 权限可收窄**（低）— 换成 9 站 host_permissions，砍掉"可读全部标签 URL"暴露面（需回归 tabs.query 调用点）。

## 三、性能

3 条候选（千问/Kimi 全 DOM 扫描、switchTier 超时翻倍）均被 2 票对抗验证否决——扫描只发生在切档短窗口内，影响被高估。无需专项，顺手优化即可（如 Kimi `querySelectorAll("*")` 收窄为 `div,li,span,button`）。

## 四、UI/UX 优化空间（按影响排序）

- ☐ **U1 失败原因不可见、不可行动**（高）— 只藏在芯片 hover tooltip，文案无指引。修：结果回齐后细条内联显示失败汇总；文案改「超时：请确认已登录且页面加载完成，点 ↻ 重试」。
- ☑ **U2 失败原因硬编码简体中文**（中）`bg/broadcast.js:91`+core.js — 泄漏到 en/zh_TW 界面，且 bg 用正则匹配中文串做流程控制。修：改错误码协议 + console 端 i18n 翻译。
- ☐ **U3 删除分组有副作用**（中）— 选中分组即 applyHosts 覆盖当前勾选。修：删除流程快照并恢复 selected。
- ☐ **U4 popup 无暗色适配**（中）— 纯白面板与暗色控制台割裂。修：CSS 变量 + `@media (prefers-color-scheme:dark)`。
- ☐ **U5 compose 缺 Ctrl+Enter 发送**（中）— 空文本/未选站也静默无反馈（与细条 shake 不一致）。
- ☐ **U6 色觉无障碍**（中）— 成功/失败只靠红绿圆点。修：fail 点加 ✕、done 加 ✓，加 aria-live 通道。
- ☐ **U7 pill 把手 tooltip 未本地化**（低）`i18n.js:88` — 三语全是旧名 "AI Model Switcher"。修：改「PolyAsk · 模型档位切换」。
- ☐ **U8 历史浏览无位置指示**（低）— 可写进 elPrompt.title「历史 n/N」。
- ☐ **U9 暗色 --faint 对比度低于 WCAG AA**（低）— 提到 #8a90a4 左右。
- ☐ **U10 zh_CN 术语不统一**（低）— popup「广播控制台」vs 其余「群发」。
- ☐ **U11 首开缺操作线索**（低）— chip title 加「点击选/取消」；「平铺」按钮 title 说明「发送会自动平铺」。
