## 一、确认的 Bug

### 🔴 高

- ☑ **B1 DeepSeek `state()` 从不读 DeepThink 开关，对话开始后恒 null** `content/adapters-cn.js:35-40` —
  只读 `[role="radio"]`，而同文件 :24 注释明确「radio 仅空对话首屏存在……档位真值由 DeepThink 开关兜底」，
  state() 却没兜。首条消息后 radio 消失：① pill 档位高亮全程熄灭；② 一键巡检 `diag_tierReadable`
  对已开聊的 DeepSeek 恒报红（误报站点失效）；③ 二轮群发 switchTier 读不到真值，退化进「两次 null
  即成功」兜底，DeepThink 点击被 DOM 重渲染吞掉的真实失败会被判成功。
  修：state() 优先读 `_deepThink()` 的 aria-pressed（开=think/关=fast），radio 仅作首屏兜底。需真机回归。
- ☑ **B2 汇总复制不转义 Markdown 特殊字符，破坏保真承诺** `content/md.js:20` —
  文本节点只折叠空白，`*`、`_`、`[`、`]`、`` ` ``、`\` 原样输出。回答含 `3*5=15`、`my_var`、`[1]`
  引用编号时，粘进 Obsidian/GitHub 被解析成强调/链接，与原文不符。table() 已对 `|` 转义（:47），
  证明意识到了、只是文本路径漏掉。修：文本节点分支追加 `.replace(/([\\`*_\[\]])/g, "\\$1")`
  （行首 `#`/`>`/`-` 场景罕见，先不做）。

  sendStart 记录、siteResult 校验）或「忽略结果直到下一次 sendStart」标志。
- ☑ **B6 retry 缺窗时空转 22s 报 timeout 而非 no_window** `bg/broadcast.js:107-124` + `console/console.js:261` —

## 三、UX 改进建议

1. **串行队列无反馈（最重要）**：openTile/sendAll/closeAll/newSession 走 serializeOp 严格排队，群发中
   （最长 ~22s）点「平铺/关闭全部/新会话」毫无反应、像卡死；「平铺」还会点击瞬间乐观置「开窗中」
   （console.js:167，实际操作还没开始）。改法：四按钮统一忙碌禁用态（send 已有先例），Tile 的 setDot
   等真正开始执行再设。
2. **「已开窗」与「已回答」共用绿勾**（status.js:22-25）：点完平铺满屏绿勾易误读为已回复。
   给 openTile 结果单独视觉态（描边而非实心勾），加一个 CSS 类即可。
3. **汇总复制反馈更诚实**（status.js:50-63）：文案只报「已复制 N 站」，不提其中 M 站是错误占位。
   copySummary 数一下 fail 数，分支文案「已复制 N 站（M 站无回答）」。
4. **历史仅键盘可达**：↑↓ 之外加一个复用 amsHistory 的原生 `<select>`（96px 约束下唯一安全形态，
   与模板下拉同款），鼠标/触屏用户才够得到。
5. **popup→console 衔接**：「打开控制台」带上当前 activeTab 的 host，console 首次无勾选历史时预勾该站，
   打通「正看着这个站、想拉几家一起问」的路径。
6. **芯片状态只在 title/颜色上**：读屏与触屏拿不到。setDot 同步更新 `aria-label`（「Claude · 已完成」）。
