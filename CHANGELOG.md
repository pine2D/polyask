# Changelog

显著变更记录，供 GitHub Release 直接引用（release.yml 提取对应版本段落作为发布说明）。
版本号与 `manifest.json` 一致；日期为发布日。

## [0.6.2] - 2026-07-06

源自一次全仓审查（8 维度深查 + 逐条对抗验证，报告见 `docs/audit-2026-07.md`）的集中修复与增强。

### 修复 · 群发可靠性（绿点不再说谎）
- contenteditable 注入失败时不再误报发送成功：注入落空立即报「注入未生效」——此前问题会静默丢失、圆点却变绿
- 豆包/Kimi/DeepSeek/千问适配器在关键控件缺失时改为报错（与其余站点契约一致），站点改版后切档失败不再弹「已切到」假提示
- 群发带档位时，档位未切换成功的站点绿点带「档位未切换，已按当前档发送」警示，深度思考对比不再被假全绿误导
- Gemini 思考等级匹配补中文界面支持，等级项找不到时报错可见，不再静默漏设

### 修复 · 窗口管理
- 根除跨 Chrome 重启的窗口 id 碰撞：启动时清空窗口登记，不再可能误关/误收编用户的 OAuth 登录弹窗等无关窗口
- 「关闭全部 / 全部新会话」与进行中的平铺/群发互斥，消除竞态——此前 Enter 群发后立刻关控制台，会再弹出整批无人管理的窗口
- 控制台置顶只认登记窗口并校验类型，console 页面被开进普通标签时不再反复抢焦日常浏览窗口
- 连按 Alt+Q（尤其后台休眠唤醒时）不再双开控制台
- 平铺基准改为控制台所在显示器：拖到哪屏铺哪屏，并修复控制台在副屏时平铺被压成 120px 废格的问题

### 修复 · 控制台交互
- 窄屏下站点芯片不再被静默裁切（改为可横向滚动），任何屏宽 9 站全部可选、状态可见
- 历史浏览（↑/↓）不再丢草稿：进入前暂存、退出时还原，浏览期间不写入存储
- 从伴侣编辑窗发起的群发失败后，「重试失败站点」按钮可用（此前恒置灰）
- 群发进行中切换分组不再抹掉各站进度圆点与失败原因
- 控制台与伴侣窗跨窗口编辑时，双向回填不再互相卡死、不再出现旧文覆盖新编辑
- 输入法兼容：分组/模板命名与历史导航不再被候选词确认（Enter）/选词（↑↓）误触发
- 悬浮胶囊从「始终显示」切回「贴边把手」后不再残留 35% 半透明

### 新增
- **一键巡检**：只读诊断所选站点适配是否失效（模型入口/档位可读等），结果直接标在芯片上，无需逐站打开 popup 手动诊断
- **会话记忆**：发送成功后记录各站会话地址，新开窗口自动续上上次会话（跨 Chrome 重启可恢复对比现场）；「全部新会话」回到空白
- **失败汇总**：全部结果回齐后，失败站点在控制台内联汇总一行（悬停看逐站原因，可一键重试）
- 只勾少数站点追问时，不再重排/关闭其他站点已生成答案的窗口（显式「平铺」按钮保持全量重排语义）

### 优化 · UI/UX 与无障碍
- 失败原因全面本地化（English/简体/繁體）：内部改错误码协议，超时等文案附带下一步指引
- popup 跟随系统暗色模式，夜间不再与暗色控制台形成白色反差
- 成功/失败状态改用 SVG 勾/叉形状标记：色觉无关可分辨，同时消除字形字符在大字体集环境下的字体回退卡顿；失败结果接入读屏播报（aria-live）
- 伴侣编辑窗支持 Ctrl/Cmd+Enter 发送，空文本/未选站有抖动反馈
- 历史浏览显示位置指示（历史 n/N）；暗色次要文字对比度提升至 WCAG AA
- 术语统一为「群发控制台」；站点芯片与平铺按钮补充首次使用提示

### 工程与安全
- 打包脚本增加产物对账断言：manifest/HTML/importScripts 引用的每个文件必须在 zip 内，缺失即打包失败（根治 v0.6.0「白名单漏项打出坏包」一类事故）
- Release workflow 的第三方 Action 固定到 commit SHA（供应链加固）
- README 警示 v0.6.1 之前 tag 的打包产物不可用
